Workflow

How the workflow is intended to run

Procedure Suite is designed to scrub PHI in the browser, then run the extraction pipeline on de-identified text. This page summarizes the current demo flow and what is still under development.

Step-by-step workflow

  1. Paste the procedural note into the editor.
  2. Run PHI detection and review each proposed redaction.
  3. Apply redactions (or manually add/remove them as needed).
  4. Submit the scrubbed note for CPT and registry extraction.
  5. Review codes, evidence, and extracted registry fields.

Current status (demo)

This dashboard is a demo. The registry data backend has not been built yet, so registry fields are placeholder outputs while we finalize the ingestion pipeline.

What happens to PHI?

PHI detection runs entirely in your browser. The server only receives the scrubbed text that you explicitly submit.
