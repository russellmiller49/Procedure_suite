# IP Adaptive Template Compiler

This compiler ingests the Interventional Pulmonology adaptive procedural DOCX and updates Reporter assets:

- Macro autogen regions in:
  - `app/reporting/templates/macros/01_minor_trach_laryngoscopy.j2`
  - `app/reporting/templates/macros/02_core_bronchoscopy.j2`
  - `app/reporting/templates/macros/03_navigation_robotic_ebus.j2`
  - `app/reporting/templates/macros/04_blvr_cryo.j2`
  - `app/reporting/templates/macros/05_pleural.j2`
  - `app/reporting/templates/macros/06_other_interventions.j2`
  - `app/reporting/templates/macros/07_clinical_assessment.j2`
- Macro metadata in `app/reporting/templates/macros/template_schema.json`
- Add-on snippets in `data/knowledge/ip_addon_templates_parsed.json`

## Source DOCX

Expected path:

- `data/template_sources/IP_Comprehensive_Adaptive_Templates.docx`

If the file is missing, the compiler exits with a clear placement error.

## Run

Check mode (no writes; non-zero exit if diffs are needed):

```bash
python ops/tools/compile_ip_adaptive_templates.py \
  --input data/template_sources/IP_Comprehensive_Adaptive_Templates.docx \
  --check
```

Write mode:

```bash
python ops/tools/compile_ip_adaptive_templates.py \
  --input data/template_sources/IP_Comprehensive_Adaptive_Templates.docx \
  --write
```

Optional debug artifacts:

```bash
python ops/tools/compile_ip_adaptive_templates.py --write \
  --emit-dir ops/tools/template_compiler/_build
```

## Overrides

Optional override file:

- `ops/tools/template_compiler/param_overrides.yaml`

Supported structures:

```yaml
by_bracket:
  "[right / left]": "side"

by_slug_ordinal:
  "ip_adaptive_s11_2_thoracentesis:1": "position"

templates:
  ip_adaptive_s11_2_thoracentesis:
    brackets:
      "[right / left]": "side"
    ordinals:
      1: "position"
```

Override precedence:

1. `by_slug_ordinal`
2. `templates.<slug>.brackets` / `by_slug_bracket`
3. `by_bracket`
4. heuristic naming

## Safe macro evolution

Generated macros are managed only inside the tagged region in each category file:

```jinja
{# === BEGIN AUTOGENERATED: ip_adaptive_templates_docx === #}
...
{# === END AUTOGENERATED: ip_adaptive_templates_docx === #}
```

Manual content outside that region is preserved.

For safe renames:

1. Add an override in `param_overrides.yaml` (for params) or adjust block title/numbering in source DOCX.
2. Run `--write`.
3. Re-run `--check` to confirm idempotency.
4. Validate downstream consumers that call macro names directly.

For new top-level DOCX sections:

1. Update section-to-category mapping in `ops/tools/template_compiler/compiler.py`.
2. Run `--write` then `--check`.
3. Run `make test`.
